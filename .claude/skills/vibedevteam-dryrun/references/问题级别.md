# 问题级别定义

## P0 - 阻塞

**定义**：无法继续开发，必须修复才能进入下一阶段。

**示例**：
- STORY 在 PRD 中没有对应需求
- STORY 需要的接口 TECH 没有定义
- TASK 验收标准完全缺失
- 循环依赖

**处理**：阻塞当前开发，必须修复

## P1 - 严重

**定义**：能继续开发，但后续会有大量返工。

**示例**：
- PRD 描述和 STORY 实现有偏差
- 接口契约不完整
- TASK 边界不清晰
- 验收标准可测试性差

**处理**：建议尽快修复，或记录风险

## P2 - 警告

**定义**：不影响当前开发，但建议优化。

**示例**：
- 文档路径引用错误
- 命名不一致
- 缺少注释

**处理**：可以后续优化，不阻塞
