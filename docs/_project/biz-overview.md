## 1. 项目愿景（Vision）

### 使命

让独立开发者和小团队能够像大厂一样，用多个 AI Dev 并行开发，提升 10 倍开发效率。

### 核心价值

1. **并行协作**：多个 AI Dev 同时工作，而不是串行排队
2. **知识沉淀**：所有决策、讨论、执行结果都可追溯
3. **标准化流程**：PRD → TECH → STORY → TASK 的标准化工作流
4. **TDD 驱动**：测试优先的开发方式，保证质量

### 目标用户

| 阶段 | 目标用户 | 核心需求 |
|-----|---------|---------|
| **Phase 1** | 独立开发者（技术能力强） | 提升个人开发效率，管理多个 AI Dev |
| **Phase 2** | 小团队（2-10 人） | 团队协作，标准化流程 |
| **Phase 3** | 企业团队 | 权限管理、审计日志、多租户 |

## 2. 项目背景（Background）

### 现状

AI 编程工具（如 GitHub Copilot、Claude Code）已经能显著提升单个开发者的效率，但**多 AI 并行协作**仍然是空白：

- **现有方案**：单 AI 串行工作（一个 Claude Code 实例）
- **问题**：无法充分利用 AI 的并行能力
- **机会**：如果能同时管理 8+ 个 AI Dev，效率可以翻倍

### 为什么是现在

1. **技术成熟**：Claude Code 提供 stream-json I/O，可以外部管理对话
2. **需求迫切**：AI 编程工具已经普及，但协作工具缺失
3. **参考先例**：Gas Town、Vibe-Kanban 等项目已经验证可行性

## 3. 项目目标与指标（Project Goals）

### 北极星指标

| 指标 | 当前值 | 目标值 | 备注 |

| 能同时管理的 AI Dev 数量 | 1 个（串行） | 8+ 个（并行） | 不增加认知负荷 |
| 开发效率提升 | 1x（baseline） | 10x | 相比单 AI 串行 |

### Phase 1 目标（验证期）

- ✅ 验证"多 AI 并行协作"的价值
- ✅ 完成核心基础设施（Beads + 工作流 + 控制台）
- ✅ 用户能舒适管理 4 个并行 dev（当前极限）

## 6. 项目级止损信号

### 技术止损

- **触发条件**：如果 Claude Code 的 stream-json 模式不稳定
- **验收标准**：连续 100 次通信测试，成功率 > 95%
- **降级方案**：退回到"一次性任务"模式

### 价值止损

- **触发条件**：如果多 AI 并行协作反而降低效率
- **验收标准**：用户报告"管理 4 个 dev 不比单 AI 效率高"
- **降级方案**：聚焦单 AI 增强，放弃多 AI 并行

### 资源止损

- **触发条件**：如果单个 Epic 开发时间超过 2 个月
- **验收标准**：每个 Epic 在 2 周内完成 MVP
- **降级方案**：缩小 Epic 范围，拆分为多个小 Epic
