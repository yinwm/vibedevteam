## 核心逻辑说明

### 1. PRD 与 TECH 的关系
*   **平行互补**: 它们是 EPIC 的左膀右臂。
    *   `PRD` 定义 **问题与范围** (In/Out)。
    *   `TECH` 定义 **模型与架构** (Database/API)。
*   **SLICE (切片)** 是它们的 **连接点**：
    *   一个 Slice (例如“登录闭环”)，需要同时引用 **PRD里的故事** 和 **TECH里的接口**。
    *   **TASK** 挂在 SLICE 下面，因为 Task 是为了完成这个 Slice 而存在的具体工作。

### 2. 身份与角色
*   我们不再使用通用的 "Dev Agent"，而是使用 **"具名实例"**。
*   **Tiger (架构师)**: 负责维护 PRD 和 TECH 文档树。
*   **Dave-1/2 (开发)**: 负责认领 TASK 并执行 `claude code`。
*   **Terry (Reviewer)**: 负责 Code Review 环节。
